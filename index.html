HTML

  • Head elements contain information about the structure of the page that are not visible on webpage
  • Title defines the title of webpage - very important for SEO (search engine optimization)
  • style define the CSS styles associated with webpage
  • "base" define base URL
  • "link" adds external resource to HTML doc
  • meta defines metadata such as the character set, description, keywords, author
  • script embeds client-side scripts in HTML doc
  • p denotes paragraph of text
  • h1, h2, etc. header texts with decreasingly sized text
  • br break between elements

CSS

    CSS notes
  • Three ways to apply CSS style to webpage
    1. Inline CSS
    2. Internal CSS
    3. External CSS
    Inline CSS
  • Used to style an element directly in the HTML file
  • Applies to specific HTML tag and only that tag
  • Quick, permanent changes
  • Less flexible than internal and external
  • i.e. using -style- to change the color of a single h1 tag
    • Internal CSS
  • Embed an entire stylesheet directly in the HTML file
  • Using -style- in -head- element
  • Simple change in CSS rule will apply to all tagged page elements
    • External CSS
  • The most common way to style a webpage
  • Other styles are acceptable only under particluar condition
  • Can store CSS rules in a separate file
  • Link CSS file using -link- element in HTML -head-
  • Declarations- contain CSS property we want to apply and the value of the property
  • Margin- space around outside of element
  • Asterisk (*) symbol applies a rule to all elements visible on a page
  • Combined two selectors into a single CSS rules with a comma
  • A period before "card" or other class type designates it as a class selector, any HTML assigned to this class will gain the properties
  • Box- shadow is described by X and Y offsets, blur and spread radius, and color

GIT


  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates new branch and switches to that branch

    Creating Feature Branch and pushing to GitHub
  1. Navigate to directory in question -cd "directory name"-
  2. Open directory in VS code -code .-
  3. Select, open file within VS Code
  4. Modify File, etc.
  5. Navigate to or confirm you are on the feature branch
  6. Commit all files to workign branch - git add -A

    7. All changes have been staged

  7. Commit changes to repository -git commit -m "message to be added with changes"-

    8. At this point all of our changes have been commited to Git repo

  8. Confirm current branch is up to date with Git repo, sync remote branch in GitHub
  9. Pull up base branch in command terminal -git pull origin main-

    10. Always pull main branch into feature branch

  10. Push up all changes -git push origin "name of branch"
  11. Navigate to repo, branch to be merged to main, pull requests
  12. Create new Pull request
  13. Compare feature branch to main branch
  14. Add title and description to PR
  15. Create & merge pull request
  16. Navigate back to main branch in GitHub
  17. Update local repo with new merged-in geature - git checkout main- - git pull origin main-
  18. Close issue on GitHub

JavaScript

    JavaScript notes
  • JavaScript adds functionality and logic to a website
  • Variables are containers that store value
  • JS is case sensitive!
  • Strings should be encased in quotation marks, whereas numbers or Boolean values should not
  • An array is a structure that allows tou to store multiple values in a single reference
  • An object is literally anything within JS, and can be stored in a variable
  • Variables are necessary to do anything dynamic in programming
  • Comments are notes added with code, marked with two slashed before comment, JS will ignore items marked as comments
  • Functions are reusable packages
  • Codes that look like a variable name, but are followed by parentheses are usually a function
  • Event Handlers are code structures that listen for activity in the browser and run code in response
  • Control Flow is the order in which a computer executes a code in a script, JS runs top to bottom
  • An Index is the location number of a piece of data, JS is zero-indexed and sequential
